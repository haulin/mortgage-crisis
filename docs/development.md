# Development notes

This repo targets the **TIC-80** fantasy console in **JavaScript**.

The runtime cartridge is `game.js`, generated by concatenating `src/*.js` in lexicographic order.

## Quick start

```bash
npm test
npm run build
```

Then in TIC-80:

```
import code game.js
```

## Source layout

- `src/` is the **source of truth**.
- Files use numeric prefixes (`00_…`, `05_…`, …) so concatenation order is deterministic and leaves gaps for future modules.
- `game.js` is **generated but committed** (treat it as a build artifact).

## JS engine compatibility (Duktape vs QuickJS)

Historically TIC-80 used **Duktape**. In Aug 2023, upstream switched to **QuickJS** (see commit [#1191: Duktape changed to QuickJS](https://github.com/nesbox/TIC-80/commit/530d7f825)).

Even though newer builds are likely QuickJS-based, the cartridge is intentionally written in an **ES5-compatible** style to stay compatible with older builds/ports and community builds.

Practical consequence: avoid modern syntax like optional chaining (`?.`) and nullish coalescing (`??`) in `src/`.

## TIC-80 reference docs (offline)

- `tic-80-docs/TIC80_API.md`: TIC-80 API reference (from official docs)
- `tic-80-docs/TIC-80.wiki/`: offline clone of the TIC-80 GitHub wiki

These are reference-only; gameplay code should not depend on them.

## Testing notes

- Tests run in Node and load `src/*.js` into a `vm` context (plus a compiled-style concat test) to catch load-order and concat issues early.
- Renderer tests stub TIC-80 draw APIs like `rect`, `spr`, etc. and assert call sequences/positions.
- Regression-first mindset: when fixing a bug, prefer adding a small assertion/test to prevent re-breaks.

## Debug harness controls (dev-only)

- `Y`: toggle DebugText ↔ Render
- In **DebugText**: `A` step, `B` next scenario, `X` reset scenario
- In **Render**: normal UI (D-pad navigate, `A` confirm/menu/target, `B` back/cancel, hold `X` for Inspect)

## Guardrails

- Don’t add runtime “shape fallbacks” like `x = x || {}` / `|| []` (enforced by tests).
- Keep numeric coercion (`|0`, `>>>0`) localized (RNG/determinism + draw-call boundaries).
- After any change in `src/` or `scripts/build.mjs`, run **both** `npm test` and `npm run build` so `game.js` doesn’t go stale.
